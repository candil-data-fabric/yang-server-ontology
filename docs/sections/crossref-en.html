Conventional Datastorec back to ToC or Class ToC

IRI: https://w3id.org/yang/server#ConventionalDatastore

One of the following set of configuration datastores: <running>, <startup>, <candidate>, and <intended>. These datastores share a common datastore schema, and protocol operations allow copying data between these datastores. The term "conventional" is chosen as a generic umbrella term for these datastores.
Is defined by
https://w3id.org/yang/server
has super-classes
Datastore c
has sub-classes
Candidate Datastore c, Intended Datastore c, Running Datastore c, Startup Datastore c

Datastorec back to ToC or Class ToC

IRI: https://w3id.org/yang/server#Datastore

A conceptual place to store and access information. A datastore might be implemented, for example, using files, a database, flash memory locations, or combinations thereof. A datastore maps to an instantiated YANG data tree.
Is defined by
https://w3id.org/yang/server
has sub-classes
Conventional Datastore c, Dynamic Datastore c, Operational Datastore c

datastore op, source datastore op

Candidate Capabilityni back to ToC or Named Individual ToC

IRI: https://w3id.org/yang/server#CandidateCapability

The candidate configuration capability, `:candidate`, indicates that the device supports a candidate configuration datastore, which is used to hold configuration data that can be manipulated without impacting the device's current configuration.
Is defined by
https://w3id.org/yang/server
belongs to

has facts
identifier dp "urn:ietf:params:netconf:capability:candidate:1.0"

Confirmed Commit Capabilityni back to ToC or Named Individual ToC

IRI: https://w3id.org/yang/server#ConfirmedCommitCapability

The `:confirmed-commit:1.1` capability indicates that the server will support the `<cancel-commit>` operation and the `<confirmed>`, `<confirm-timeout>`, `<persist>`, and `<persist-id>` parameters for the `<commit>` operation.
Is defined by
